Conceptos del laboratorio:

  • URL: Conjunto de carácteres que son usados como identificación de la ubicación de un sitio web. [1]
  • MIME: Manera standard de mandar contenido a través de la red. [2]
  • Métodos y códigos de errores de HTTP: Describen de forma abreviada la respuesta de consultas HTTP. [3]
  • XML: Es un lenguaje de marcado para facilitar la definición de lenguajes para diferentes necesidades. [4]
  • XHTML: HTML escrito en forma de XML. [5]
  • HTML 5: Es la última versión de HTML, realizando un conjunto amplio de tecnologías que permite que un sitio web sea más diverso. [6]
  • JS: Es un lenguaje orientado a objetos usado comunmente en páginas web. [7]
  • PHP: Es un lenguaje interpretado en páginas HTML y ejecutado en el servidor. [8]
  • JSP: Tecnología orientada a crear páginas web con programación en Java. [9]
  • AJAX: Modo de utilizar conjuntamente teconologías como XHTML, CSS, Js, XML, entre otros. Esto para eficientar el rendimiento del sitio web. [10]

Preguntas a responder:

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?Internet es la red que existe entre los dispositivos alrededor del mundo, www es la colección de páginas que existen en el internet.
  2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?Get: solicita una representación de un recurso específico. HEAD: pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta. POST: se utiliza para enviar una entidad a un recurso en específico. PUT: reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición. PATCH: es utilizado para aplicar modificaciones parciales a un recurso. DELETE: borra un recurso en específico.
  3. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? GET
  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? POST, porque debes enviar información.
  5. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? Significa que la solicitud ha tenido éxito, no ocurrió ningún error.
  6. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? No corregir el sitio web, ya que la falla no está ahí. La falla se encuentra en el servidor.
  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? No es error del sitio web, es error del servidor, por lo tanto no le corresponde al desarrollador
  8. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Significa que esas funciones o características han dejado de recibir soporte y han sido sustituidas o se ha desarrollado otra característica o función más actualizada. Algunos ejemplos son: BASEFONT, CENTER, S, STRIKE.
  9. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5? Sintaxis, especialmente la compatibilidad con XML a través de XHTML. HTML5 permite codificación de carateres. HTML5 requiere el elemente doctype. HTML5 tiene nuevos elementos como: article, aside y dialog.
  10. ¿Qué componentes de estructura y estilo tiene una tabla? se compone por "rows" y por celdas "data".
  11. ¿Cuáles son los principales controles de una forma HTML5? cajas de texto, cajas de texto para claves, botones de elección, cajas de selección, botón de borrado/cancelar, botónes.
  12. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas? Puedes utilizar la siguiente página para descubrirlo: http://html5test.com/ (Al responder la pregunta recuerda poner el navegador que utilizas)Opera: 481 de 555 puntos
